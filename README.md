# AI 产品经理手册

这是一本写给 AI 产品经理的实战手册。

不讲概念，讲经验。不讲"AI 是什么"，讲"做 AI 产品时最容易在哪里翻车"。目标是帮你从零建立起真正有战斗力的 AI 产品认知，而不只是能和工程师对话。

---

## 为什么写这本书

市面上不缺 AI 科普。缺的是：把真正做过 AI 产品的人踩过的坑、做出过的判断、用过的框架，原原本本讲清楚。

我在腾讯做过 AI 产品组，负责把大模型落地到 QQ 的各个场景，从零搭建过评估体系和 Agent 平台；在 Keep 负责 Keepace.ai 大模型和 Kaka Agent 教练。这些年下来，我见过太多 PM 卡在同一个地方——不是不够努力，而是没有人告诉他们，在 AI 产品里，哪些事情是真正重要的，哪些是在浪费时间。

这本手册是我把这些经验系统化的一次尝试。

它不讲概念，讲经验。不讲"你应该了解大模型"，讲"一个大模型在你的产品里出了问题，你该怎么诊断"。

---

## 这本书适合谁

- 在做 AI 产品或负责 AI 功能的产品经理
- 想从传统 PM 转型为 AI PM 的人
- 需要和算法、工程团队深度协作的 PM
- 想系统建立 AI 产品认知框架的人

一个真正有战斗力的 AI PM，在我看来需要做到这些：

- 深度理解大模型的工作方式，不只是知道 token 是什么，而是能判断一个问题是模型的问题还是工程的问题
- 能独立搭建简单的 Agent，至少要能用 LangGraph 跑通一个 ReAct 流程，而不是只听说过
- 能设计评测集、定义验收标准，而不是把"效果好不好"这个问题全部交给算法同学
- 碰到意图识别、记忆管理、工具调用这些问题，能讲出自己的判断，而不只是传话

这个标准，比大多数人预期的要高。这本书的目标，就是帮你达到这个标准。

不要求你现在已经会。但读完这本书，你应该知道自己在哪里、差什么、怎么补。

## 内容结构

**第零章：AI PM 的新起点** — 重建认知框架，明确定位，建立工具箱

**第一章：先学会衡量** — 评测体系的建立，这是 AI PM 最容易忽视、也最容易翻车的地方

**第二章：大模型是怎么造出来的** — 训练流程、对齐算法、硬件、数据、监控，建立完整的训练认知

**第三章：大模型是怎么工作的** — 推理过程、采样控制、Prompt Engineering、API 调用，从理解到动手

**第四章：安全与治理** — 内容安全、数据合规、国内备案与监管，PM 的责任边界在哪

**第五章：Agent 全景** — 从历史到现状，意图、记忆、编排、框架、Skill 生态

**第六章：动手做 Agent** — 以真实业务为例，用 LangGraph 走完完整的落地流程

**第七章：大模型时代的商业化** — 成本结构、ROI、Token 付费逻辑、定价策略
