- AI PM 的工具箱：平时用什么、怎么用、为什么要用
  - 模型体验与对比：ChatGPT、Claude、Gemini、Kimi——每天要用，建立手感
  - API 调试：OpenAI Playground、Anthropic Console，直接感受参数变化
  - 编程环境：Jupyter Notebook、Google Colab，跑通代码不依赖工程师
  - AI 编程助手：Cursor——AI PM 自己写代码的正确工具
  - Agent 搭建与原型验证：Dify、Coze，快速跑通业务 demo
  - 流程与链路调试：Langfuse，看清 Agent 每一步在做什么
  - 数据与分析：SQL + Python 是基本功，不会的先补上
  - 信息源管理：如何持续跟踪 AI 行业动态，不被信息噪音淹没

## 第二章：理解大模型（上）——它是怎么学会的

- 预训练、SFT、RLHF：每一步在做什么，解决什么问题
- 领域大模型的训练过程：PM 能参与什么、贡献什么
- 行业大模型发展历程：从 GPT-1 到现在，每一步的意义
- 主流模型盘点：各家模型的能力边界与适用场景
- Tokenization 与 BPE：模型是怎么"读"文字的，为什么这很重要
  - BPE（Byte Pair Encoding）：分词逻辑与词表构建
  - 不同语言的 token 效率差异对产品成本的影响
- 后训练对齐算法的演进：从 PPO 到 GSPO
  - PPO：RLHF 的基石，四模型架构，为什么它贵且难训练
  - DPO：绕过奖励模型，直接对比偏好，简化了什么、丢掉了什么
  - GRPO：DeepSeek 的突破，去掉 Critic，组内相对比较的思路
  - DAPO：动态采样 + 解耦裁剪，解决长文本训练不稳定问题
  - GSPO：从 Token 级到序列级优化，更适配 MoE 架构和长文本
  - RLVR：用可验证奖励替代人类标注，自动化对齐的新方向
  - 演进趋势：从复杂到精简，从人工标注到自动验证，从静态到动态
- 硬件基础：算力从哪来，PM 需要知道什么
  - 为什么是 GPU：并行计算的本质，和 CPU 的根本区别
  - CUDA 生态的意义：为什么英伟达的护城河那么深
  - 英伟达主流 GPU 盘点：A100、H100、H200、B100 的能力与定价差异
  - 训练卡 vs 推理卡：为什么训练和推理对硬件的要求不同
  - 国产 GPU 玩家：华为昇腾、寒武纪、摩尔线程、壁仞科技的现状
  - 国内云平台算力服务：阿里云 PAI、火山引擎 MLP、腾讯云 TI、百度智能云
  - 低成本算力方案：AutoDL、揽睿星舟、闲置 GPU 租赁，穷鬼也能训模型
- 训练数据从哪来：数据集平台与常用资源
  - Hugging Face Datasets：最大的开源数据集社区
  - 魔搭社区（ModelScope / Alibaba）：国内最全的模型与数据集平台
  - 常见开源数据集类型：预训练语料、指令微调集、偏好对齐数据
  - 数据质量 vs 数据规模：为什么数据是训练的真正瓶颈
- 从业务中挖掘训练数据：真正的竞争壁垒
  - 为什么业务数据比开源数据更有价值
  - 数据挖掘 Pipeline 设计：从原始日志到可用训练样本的全流程
    - 数据源识别：用户行为日志、对话记录、内容库、人工产出在哪里
    - 数据清洗：去重、去噪、过滤低质量样本的策略
    - 数据增强：如何用少量高质量样本扩展训练集
    - 数据飞轮：线上效果反哺训练数据的闭环机制
  - 数据标注：如何把原始数据变成可训练的样本
    - 标注任务设计：指令数据、偏好对比数据、拒绝采样数据的标注差异
    - 标注平台：LabelLLM（开源，专为大模型标注设计）及同类工具
    - 标注质量控制：一致率、仲裁机制、标注者校准
    - 用大模型辅助标注：合成数据与人工标注的边界在哪
  - 数据处理工具
    - Data-Juicer（阿里开源）：面向大模型训练的数据处理算子库
    - 常用算子类型：文本质量过滤、去重、格式转换、多样性采样
    - 如何评估数据集质量：分布分析、覆盖度检验、毒性检测
- 训练过程监控：怎么知道模型在往好的方向走
  - 核心指标：Training Loss、Validation Loss、学习率曲线、梯度范数
  - Loss 曲线的常见形态：正常收敛、过拟合、训练崩溃的识别
  - Reward 与 KL 散度：对齐训练阶段的专属指标
  - 可视化工具：Weights & Biases（W&B）的核心功能与使用场景
  - 轻量替代：TensorBoard、SwanLab（国产 W&B 替代）
  - PM 看监控的正确姿势：哪些曲线你需要关注，哪些可以交给算法

## 第三章：理解大模型（下）——它是怎么工作的，以及边界在哪

- 推理过程：token 生成的本质
  - Temperature：控制随机性，高低温的产品意义
  - Top-K：只从概率最高的 K 个 token 中采样
  - Top-P（Nucleus Sampling）：动态截断，为什么比 Top-K 更常用
  - Repetition Penalty、Frequency Penalty：控制重复的手段
  - 这些参数怎么影响产品体验：创意写作 vs 事实问答的不同配置策略
- 位置编码：模型怎么理解"顺序"
  - RoPE（Rotary Position Embedding）：旋转位置编码的直觉理解
  - YaRN：上下文窗口扩展的方案，长文本能力从哪来
- 如何影响大模型效果：Prompt、RAG、微调、蒸馏各自的边界
- Prompt Engineering 实战
  - 基础技巧：Zero-shot、Few-shot、CoT（思维链）的适用场景
  - System Prompt 设计：角色、约束、输出格式的分层控制
  - 结构化输出的陷阱：为什么不能直接让大模型输出复杂 JSON
  - 输出协议压缩：用自定义紧凑格式替代 JSON，再用解析函数还原
    - 原理：减少 token 消耗、降低格式错误率、提升输出稳定性
    - 实战案例：Keep 训练计划生成的压缩协议设计（`#str` 管道符格式）
    - 设计原则：如何为自己的业务设计一套压缩协议
  - Prompt 的版本管理与回归测试：工程化 Prompt 的正确姿势
- 大模型的上限：幻觉从哪来，能力天花板怎么理解
- 大模型产品指标设计：延迟、成本、质量的三角权衡
- 主流 API 协议与对话格式：你调用的接口背后是什么
  - 7 行代码：每个 AI PM 必须能默写的 OpenAI 标准调用
    - 这 7 行在做什么：client、model、messages、role、content、response
    - 为什么要求默写：不是为了当工程师，是为了建立真实的技术直觉
  - OpenAI、Claude（Anthropic）、Gemini（Google）的协议差异
  - ChatML 与各家消息格式的异同
  - System Prompt、User/Assistant 角色设计对产品的意义
  - 协议差异对多模型切换、成本控制的实际影响

## 第五章：Agent 全景——从历史到工业现状

- Agent 的发展历程：从 ReAct 到 MAS 到 Agentic AI
- 主流 Agent 框架盘点与选型逻辑
- 典型 Agent 产品案例分析：它们的设计逻辑是什么
- 不同业务场景下的 Agent 形态（待补充）
- 框架选型逻辑：什么场景用什么，不要盲目跟风
- Agent 的能力扩展：Skill、CLI、MCP 是什么，产品意义在哪
  - Skill 深度解析
    - Skill 是什么：Agent 的可复用能力单元，和工具调用的区别
    - Skill 的开发流程：从需求定义到发布的完整链路
    - Skill 的开发方式：函数式、服务化、模型驱动的不同实现路径
    - Skill 的数据指标：调用量、成功率、延迟、用户满意度的度量体系
    - Skill 的分发平台：生态在哪里，如何选择和发布
      - npm：通用包管理，JS 生态的 Skill 分发
      - ClawHub、SkillHub、SkillsMP：专为 AI Skill 设计的分发平台
      - skills.sh：命令行风格的 Skill 管理工具
      - ModelScope（魔搭）：国内主流，模型与 Skill 一体化生态
    - Skill 生态的产品意义：为什么 Skill 市场会成为新的竞争阵地
  - CLI：让 Agent 具备终端执行能力意味着什么
  - MCP（Model Context Protocol）：标准化工具调用协议的产品价值
  - 这些技术对产品边界的影响：Agent 能做的事为什么越来越多
- Agent Orchestration Framework & Runtime：编排层的核心概念
  - Durable Execution：为什么 Agent 需要持久化执行，宕机、重试、断点续跑
  - Human-in-the-loop：哪些环节必须人介入，如何设计审批与中断机制
  - Stateful Workflow：有状态工作流与无状态调用的根本区别
  - Production Deployment：Agent 上生产环境面临的真实挑战
- Agent 的认知能力：意图、记忆、推理
  - 意图识别：从用户输入到 Agent 行动的第一步
    - 意图分类的常见方案：分类器 vs 大模型直接判断
    - 意图识别的准确率瓶颈：歧义、多意图、隐式意图
    - 如何设计意图体系：槽位、置信度、兜底策略
  - 记忆的三个层次
    - 短期记忆（In-context Memory）：当前对话窗口内的上下文，容量有限
    - 中期记忆（Working Memory）：跨轮次但同会话的任务状态与中间结果
    - 长期记忆（Persistent Memory）：跨会话、跨用户的知识沉淀与偏好积累
  - 记忆的工程实现
    - 向量数据库做语义检索：什么时候该用、什么时候不该用
    - mem0：专为 Agent 设计的记忆管理框架，层次化存储与自动提取
    - 其他记忆方案：Zep、MemGPT 的思路对比
    - 记忆的产品设计：用户该不该感知到 Agent 在"记住"他
