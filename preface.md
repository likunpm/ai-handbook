# 前言

## 为什么写这本书

市面上不缺 AI 科普。缺的是：把真正做过 AI 产品的人踩过的坑、做出过的判断、用过的框架，原原本本讲清楚。

我在腾讯做过 AI 产品组，负责把大模型落地到 QQ 的各个场景，从零搭建过评估体系和 Agent 平台；在 Keep 负责 Keepace.ai 大模型和 Kaka Agent 教练。这些年下来，我见过太多 PM 卡在同一个地方——不是不够努力，而是没有人告诉他们，在 AI 产品里，哪些事情是真正重要的，哪些是在浪费时间。

这本手册是我把这些经验系统化的一次尝试。

它不讲概念，讲经验。不讲"AI 是什么"，讲"AI 产品里最容易出问题的地方在哪"。不讲"你应该了解大模型"，讲"一个大模型在你的产品里出了问题，你该怎么诊断"。

---

## 这本书的目标读者

一个真正有战斗力的 AI 产品经理，在我看来需要做到这些：

- 深度理解大模型的工作方式，不只是知道 token 是什么，而是能判断一个问题是模型的问题还是工程的问题
- 能独立搭建简单的 Agent，至少要能用 LangGraph 跑通一个 ReAct 流程，而不是只听说过
- 能设计评测集、定义验收标准，而不是把"效果好不好"这个问题全部交给算法同学
- 碰到 MAS 里的意图识别、记忆管理、工具调用这些问题，能讲出自己的判断，而不只是传话

这个标准，比大多数人预期的要高。

这本书的目标，就是帮你达到这个标准。

不要求你现在已经会。但读完这本书，你应该知道自己在哪里、差什么、怎么补。

## 怎么读这本书

五个部分，建议顺序读，特别是第一部分不要跳过。

**第一部分：AI 与算法**——从机器学习的起点讲到大语言模型、多模态、世界模型，以及如何从零构建一个模型。这部分会有一定深度，目标不是让你背公式，而是让你建立真正准确的直觉，能看穿表象。

**第二部分：AI 产品的特点**——AI 产品和传统软件产品有哪些根本不同，这些不同对设计、对验收、对用户信任意味着什么。

**第三部分：Agent 框架与工业落地**——覆盖当下真实在用的框架和产品形态，包括 RAG、Copilot、LUI Chatbot、Cursor、OpenClaw、Hermes，以及你必须掌握的 LangGraph。这部分尽量结合真实案例，不讲架构图，讲实际跑起来的系统长什么样。

**第四部分：评估与度量**——AI 产品的质量该怎么量化，评测集怎么设计，人工评估里有哪些系统性偏差。这是很多 PM 最容易忽视、但又最容易翻车的地方。

**第五部分：安全与治理**——内容安全、数据合规、偏见问题，以及国内外监管的现状。作为 PM，你需要知道自己的责任边界在哪。
