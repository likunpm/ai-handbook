## 第二章：大模型是怎么造出来的

> 不要求你会训练，但你必须知道它经历了什么

### 2.1 训练流程
- 预训练：在互联网上学语言的本质是什么
- SFT（指令微调）：从"会说话"到"能干活"
- RLHF：从"能干活"到"符合人类偏好"
- 领域大模型训练：PM 能参与什么、贡献什么价值

### 2.2 后训练对齐算法的演进
- PPO：RLHF 的基石，四模型架构，为什么贵且难
- DPO：绕过奖励模型，简化了什么、丢掉了什么
- GRPO：DeepSeek 的突破，去掉 Critic，组内相对比较
- DAPO：动态采样 + 解耦裁剪，解决长文本不稳定
- GSPO：Token 级 → 序列级，更适配 MoE 和长文本
- RLVR：可验证奖励替代人工标注，自动化对齐的新方向
- 演进趋势：从复杂到精简，从人工到自动，从静态到动态

### 2.3 Tokenization 与 BPE
- BPE 的分词逻辑：模型是怎么"读"文字的
- 不同语言的 token 效率差异对产品成本的影响

### 2.4 算力与硬件
- 为什么是 GPU：并行计算的本质
- CUDA 生态的意义：英伟达护城河从哪来
- 主流 GPU 代际：A100、H100、H200、B100 的能力差异
- 训练卡 vs 推理卡：需求为什么不同
- 国产 GPU 现状：华为昇腾、寒武纪、摩尔线程、壁仞
- 国内云平台：阿里云 PAI、火山引擎 MLP、腾讯云 TI、百度智能云
- 低成本方案：AutoDL、揽睿星舟等租赁平台

### 2.5 训练数据
- Hugging Face Datasets、魔搭社区（ModelScope）
- 开源数据集类型：预训练语料、SFT 指令集、偏好对齐数据
- 数据质量 vs 数据规模：为什么数据才是真正的瓶颈
- 从业务中挖掘训练数据：真正的竞争壁垒
  - 为什么业务数据比开源数据更有价值
  - 数据挖掘 Pipeline：从原始日志到可用训练样本的全流程
    - 数据源识别：用户行为、对话记录、内容库、人工产出
    - 数据清洗：去重、去噪、过滤低质量样本
    - 数据增强：用少量高质量样本扩展训练集
    - 数据飞轮：线上效果反哺训练数据的闭环机制
  - 数据标注
    - 标注任务设计：指令数据、偏好对比、拒绝采样数据的差异
    - 标注平台：LabelLLM（开源，专为大模型标注设计）及同类工具
    - 标注质量控制：一致率、仲裁机制、标注者校准
    - 用大模型辅助标注：合成数据与人工标注的边界
  - 数据处理工具
    - Data-Juicer（阿里开源）：面向大模型的数据处理算子库
    - 常用算子：文本质量过滤、去重、格式转换、多样性采样
    - 如何评估数据集质量：分布分析、覆盖度、毒性检测

### 2.6 训练过程监控
- 核心指标：Training Loss、Validation Loss、学习率曲线、梯度范数
- Loss 曲线识别：正常收敛、过拟合、训练崩溃
- 对齐阶段专属：Reward 曲线、KL 散度
- 工具：W&B、TensorBoard、SwanLab
- PM 看监控的正确姿势：哪些你该关注，哪些交给算法

### 2.7 行业大模型盘点
- 从 GPT-1 到现在：每一步的意义
- 主流模型能力边界与适用场景对比

## 第三章：大模型是怎么工作的

> 从调用到理解，从参数到边界

### 3.1 先跑起来：7 行代码
- 每个 AI PM 必须能默写的 OpenAI 标准调用
- 这 7 行在做什么：client、model、messages、role、content、response
- 为什么要求默写：建立真实的技术直觉，不是为了当工程师

### 3.2 主流 API 协议与对话格式
- OpenAI、Claude（Anthropic）、Gemini（Google）的协议差异
- ChatML 与各家消息格式的异同
- System Prompt、User/Assistant 角色设计的产品意义
- 协议差异对多模型切换、成本控制的实际影响

### 3.3 推理过程与采样控制
- Token 生成的本质：模型在做什么
- Temperature：控制随机性，高低温的产品意义
- Top-K 与 Top-P（Nucleus Sampling）：有什么区别，为什么 Top-P 更常用
- Repetition Penalty、Frequency Penalty：控制重复的手段
- 不同场景的参数配置策略：创意写作 vs 事实问答

### 3.4 位置编码与上下文窗口
- RoPE：旋转位置编码的直觉理解
- YaRN：上下文窗口扩展，长文本能力从哪来

### 3.5 如何影响大模型效果
- Prompt Engineering、RAG、微调、蒸馏——各自的边界在哪
- 什么时候该用哪种，不要乱选

### 3.6 Prompt Engineering 实战
- Zero-shot、Few-shot、CoT 的适用场景
- System Prompt 设计：角色、约束、输出格式的分层控制
- 结构化输出的陷阱：为什么不能直接让模型输出复杂 JSON
- 输出协议压缩：自定义紧凑格式 + 解析函数的工程思路
  - 实战案例：Keep 训练计划生成的 `#str` 压缩协议
  - 设计原则：如何为自己的业务设计一套压缩协议
- Prompt 的版本管理与回归测试

### 3.7 大模型的上限与产品指标
- 幻觉从哪来，能力天花板怎么理解
- 延迟、成本、质量的三角权衡
- 大模型产品的核心指标设计

## 第五章：Agent 全景

> 知道这个领域在哪，谁在做什么，底层逻辑是什么

### 5.1 Agent 发展历程
- 从 ReAct 到 MAS 再到 Agentic AI
- 每个阶段解决了什么问题，引入了什么新挑战

### 5.2 Agent 的认知能力
- 意图识别：从用户输入到 Agent 行动的第一步
  - 分类器 vs 大模型直判的方案对比
  - 准确率瓶颈：歧义、多意图、隐式意图
  - 意图体系设计：槽位、置信度、兜底策略
- 记忆的三个层次
  - 短期记忆（In-context）：对话窗口内，容量有限
  - 中期记忆（Working Memory）：跨轮次，同会话任务状态
  - 长期记忆（Persistent）：跨会话，知识沉淀与用户偏好
- 记忆的工程实现
  - 向量数据库的适用边界
  - mem0：专为 Agent 设计的记忆管理框架
  - Zep、MemGPT 的思路对比
  - 产品设计问题：用户该不该感知到 Agent 在"记住"他

### 5.3 Agent 的能力扩展
- Skill 深度解析
  - Skill 是什么：Agent 的可复用能力单元，和工具调用的区别
  - Skill 的开发流程：从需求定义到发布的完整链路
  - Skill 的开发方式：函数式、服务化、模型驱动的不同实现路径
  - Skill 的数据指标：调用量、成功率、延迟、用户满意度
  - Skill 的分发平台
    - npm：JS 生态的通用分发
    - ClawHub、SkillHub、SkillsMP：专为 AI Skill 设计的分发平台
    - skills.sh：命令行风格的 Skill 管理工具
    - ModelScope（魔搭）：国内模型与 Skill 一体化生态
  - Skill 生态的产品意义：为什么 Skill 市场会成为新的竞争阵地
- CLI：让 Agent 具备终端执行能力意味着什么
- MCP（Model Context Protocol）：标准化工具调用协议的产品价值

### 5.4 Orchestration & Runtime
- Durable Execution：持久化执行，断点续跑
- Human-in-the-loop：哪些环节必须人介入，审批与中断机制
- Stateful Workflow：有状态 vs 无状态的根本区别
- Production Deployment：上生产环境面临的真实挑战

### 5.5 主流框架与典型产品
- 主流 Agent 框架盘点：LangGraph、AutoGen、CrewAI、Dify、Coze 等
- 典型产品案例分析：Cursor、Devin、Perplexity 等的设计逻辑
