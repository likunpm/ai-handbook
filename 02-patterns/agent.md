# Agent

Agent 一般指能够围绕目标自主完成一系列步骤的 AI 系统。

和单次问答相比，Agent 更强调：

- 目标驱动
- 多步规划
- 工具调用
- 状态管理
- 持续执行

## 为什么 Agent 受到关注

因为很多真实工作不是“一问一答”，而是一个流程：

- 理解目标
- 拆解任务
- 查找信息
- 调用工具
- 产出结果
- 根据反馈继续修正

Agent 试图把这些步骤组织起来。

## 一个简单例子

如果目标是“整理竞品调研”，一个 Agent 可能会：

1. 搜集目标公司资料
2. 提取产品功能
3. 对比差异
4. 生成报告草稿
5. 根据要求继续修改

## Agent 不等于完全自动化

现实中很多 Agent 更适合做“半自动协作系统”，而不是完全脱离人类监督。

原因包括：

- 工具调用可能失败
- 任务拆解可能不稳定
- 长链路会累积误差
- 某些环节仍然需要人判断

## 一句话理解

Agent 是把模型从“回答者”升级为“执行者”的一种系统设计方式。
