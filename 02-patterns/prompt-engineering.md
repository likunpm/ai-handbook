# Prompt Engineering

Prompt Engineering 指的是通过设计输入方式，让模型更稳定地产生期望输出。

## 为什么需要 Prompt

大模型不是传统 API。你不能只给它一个函数名，就期待它永远稳定执行。

Prompt 的作用，是把任务边界、角色、格式、限制条件和成功标准表达清楚。

## 一个好 Prompt 通常包含什么

- 任务目标
- 背景上下文
- 输出格式
- 限制条件
- 示例

## Prompt 的意义

Prompt 不是“咒语”，而是一种任务表达方式。

它的重要性在于：

- 降低试错成本
- 提高输出一致性
- 让业务意图更容易转化为模型行为

## 它的边界

Prompt 可以显著改善效果，但不能解决所有问题。

如果问题来自：

- 缺少业务知识
- 缺少实时数据
- 工具调用能力不足
- 模型本身能力不够

那么仅靠 Prompt 往往不够，需要引入 RAG、工具、工作流编排或更强模型。
