# Fine-tuning

Fine-tuning，微调，是在已有基础模型上继续训练，使其更适应特定任务、风格或领域数据。

## 为什么要微调

并不是所有问题都要微调，但在一些场景里它很有价值，例如：

- 特定领域术语很多
- 输出风格要求高度一致
- 有足够高质量样本
- 单靠 Prompt 难以稳定达到目标

## 微调和 RAG 的区别

一个常见误区是把两者混为一谈。

它们解决的问题不同：

- RAG 主要解决“模型不知道这些资料”
- 微调主要解决“模型行为方式不够贴合任务”

简单说：

- 缺知识，优先考虑 RAG
- 缺风格或行为一致性，考虑 Fine-tuning

## 微调的代价

- 需要数据清洗和标注
- 需要训练与评估成本
- 版本管理更复杂
- 可能引入新的偏差

因此，是否微调要看收益是否明显大于维护成本。
