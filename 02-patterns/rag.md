# RAG

RAG，Retrieval-Augmented Generation，中文常译为“检索增强生成”。

它的核心思想是：在模型回答前，先从外部知识源检索相关信息，再把这些信息连同问题一起提供给模型生成答案。

## 为什么需要 RAG

大模型本身并不天然拥有：

- 企业私有知识
- 最新资料
- 高精度事实保证

RAG 的价值就在于把“模型能力”和“外部知识”连接起来。

## 一个典型流程

1. 用户提出问题
2. 系统把问题转换为检索请求
3. 从知识库中找出相关内容
4. 把检索结果连同问题交给模型
5. 模型基于上下文生成答案

## 它适合什么场景

- 企业知识库问答
- 文档助手
- 客服辅助
- 研报、制度、产品资料查询

## 它解决的核心问题

- 减少幻觉
- 引入私有知识
- 增强可追溯性
- 让答案更贴近具体业务语境

## 需要注意什么

RAG 的质量不只取决于模型，也取决于：

- 文档切分是否合理
- 检索是否准确
- 重排序是否有效
- Prompt 是否能正确使用检索结果

因此，RAG 本质上是一个系统工程，不只是“接一个向量库”。
