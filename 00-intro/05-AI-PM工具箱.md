# 5. AI PM 的工具箱：从模型体验到链路调试，你每天该用什么

> 工具不是目的，但没有合适的工具，很多事做不了。这里列的，是一个 AI PM 每天真正会用到的东西。

---

## AI 时代的产品需求文档

先说一个很多人忽略的变化：**AI 产品的需求文档写法和传统 PRD 不一样。**

传统 PRD 的核心是：用例描述、界面原型、交互说明、边界条件。

AI 产品的需求文档需要额外包含：

- **任务定义**：这个 AI 功能要完成的具体任务是什么（不是"智能回复"，而是"根据用户的运动记录，生成一个本周总结，包含完成率、亮点和下周建议"）
- **输入输出规格**：输入是什么格式，输出期望是什么格式，有什么约束
- **评测标准**：什么叫"好"，怎么量化，谁来判断
- **失败案例预设**：已知的难点和边界情况是什么
- **Prompt 草稿**：PM 要能给出第一版 Prompt，不能全部扔给算法

如果你还在用写传统功能的方式写 AI 需求，大概率会在对齐上浪费大量时间。

## 工具 1：强大的 AI 协同伙伴

**推荐：Cursor / Claude Code / Codex**

这不是"AI 写代码帮你偷懒"，而是**让你能动手的基础设施**。

作为 AI PM，你经常需要：
- 快速验证一个 Prompt 是否有效
- 跑一段数据处理脚本，看看分布是什么样
- 搭一个最小可运行的 Agent demo
- 调一个 API，看看返回值长什么样

这些事情，过去需要找工程师帮你做，现在你可以自己在 10 分钟内完成。

**推荐使用方式：**
不要用来"生成完整项目"，要用来"快速验证想法"。把你的问题描述清楚，让 AI 帮你写第一个版本，你来看结果、提问题、迭代。

---

## 工具 2：本地 Jupyter 环境

**推荐：JupyterLab + Python**

实践是最好的老师。一个本地的 Jupyter 环境，让你可以：

- 直接调用大模型 API，测试不同 Prompt 的输出
- 做数据分析，看评测集的分布、错误集中在哪里
- 跑评测脚本，计算 Rubric 分数
- 探索新的框架（LangGraph、LlamaIndex……）

**快速搭建方式：**

```bash
pip install jupyterlab openai langchain
jupyter lab
```

装好之后，第一件事：用 7 行代码跑通一个大模型 API 调用（这是第三章的作业，但越早做越好）。

---

## 工具 3：一个云服务账号

**推荐：阿里云 / 腾讯云 / Vercel（看你的场景）**

本地跑通之后，你需要一个地方把东西发布出去。

为什么 PM 需要会部署？

因为很多判断只有在真实环境里才能做——本地测试没有延迟压力，没有并发，没有真实用户的输入多样性。把一个 demo 发出去，让真实用户用，你能得到在本地永远看不到的问题。

**最低门槛方案：**
- 用 Vercel 部署一个简单的 web 界面，接上大模型 API
- 或者用 Gradio / Streamlit 快速做一个可分享的原型

不需要会运维，重点是能把东西放到公网上让人访问。

---

## 工具 4：大模型 API Key

**推荐同时持有：OpenAI / Anthropic Claude / 国内主流模型（通义/智谱/DeepSeek）**

最好的学习方法是实践。有了 API Key，你才能：

- 亲手测试不同模型在同一任务上的表现差异
- 感受不同模型的延迟和成本
- 理解 System Prompt 在不同模型上的行为差异
- 建立对"这个任务该用哪个模型"的真实直觉

**建议的入门路径：**

1. 注册 OpenAI 账号，拿到 API Key
2. 用 Python 跑通 7 行调用代码
3. 用同一个 Prompt 测试 temperature = 0.1 / 0.7 / 1.5 的输出差异
4. 换一个国内模型，看同样的 Prompt 表现有什么不同

这个过程不需要花很多钱，但它会给你建立起对大模型行为的基本直觉——这个直觉，是后续所有工作的基础。

## 工具体系总结

| 工具 | 解决什么问题 | 优先级 |
|---|---|---|
| AI 编程助手（Cursor/Claude Code） | 让你能动手验证想法 | ⭐⭐⭐⭐⭐ |
| 本地 Jupyter 环境 | 快速实验和数据分析 | ⭐⭐⭐⭐⭐ |
| 云服务账号 | 把 demo 发出去，接触真实场景 | ⭐⭐⭐⭐ |
| 大模型 API Key | 建立对模型行为的直觉 | ⭐⭐⭐⭐⭐ |

这些工具加在一起，构成了一个 AI PM 的最小工具集。不是为了"看起来很技术"，而是为了让你在做产品决策时，有足够的动手能力来支撑你的判断。

## 本章作业

选 3 个大模型产品，用同一个复杂任务测试（比如："帮我制定一个适合上班族的健身计划，每周 3 次，每次 45 分钟，目标是减脂"），记录：

1. 各自的输出内容差异
2. 你觉得哪个更好，好在哪里（具体到某一句话或某种结构）
3. 如果你是这三个产品的 PM，你会优先改进哪个问题
